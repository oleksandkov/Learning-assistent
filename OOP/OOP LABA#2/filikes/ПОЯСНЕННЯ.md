# Детальне пояснення Лабораторної роботи #2

## Загальний опис

Програма демонструє основні принципи об'єктно-орієнтованого програмування (ООП):
- Створення класів та об'єктів
- Інкапсуляція (приватні поля)
- Конструктори
- Геттери та сеттери
- Методи класу

Програма реалізує систему управління студентами, де кожен студент представлений як окремий об'єкт класу `Student`.

---

## Структура програми

### 1. Клас Student

```java
class Student {
    // Поля, конструктор, методи
}
```

**Пояснення:** Клас - це шаблон для створення об'єктів. Клас `Student` описує властивості та поведінку студента.

---

### 2. Приватні поля (Інкапсуляція)

```java
private String name;
private String surname;
private int course;
private double averageGrade;
private String faculty;
```

**Пояснення:**
- `private` - модифікатор доступу, який робить поля недоступними ззовні класу
- **Інкапсуляція** - приховування внутрішніх даних об'єкта
- Доступ до полів можливий лише через спеціальні методи (геттери/сеттери)

**Чому це важливо:**
- Контроль над даними (можна перевіряти значення перед зміною)
- Безпека даних
- Гнучкість (можна змінити реалізацію без зміни інтерфейсу)

---

### 3. Конструктор

```java
public Student(String name, String surname, int course, double averageGrade, String faculty) {
    this.name = name;
    this.surname = surname;
    this.course = course;
    this.averageGrade = averageGrade;
    this.faculty = faculty;
}
```

**Пояснення:**
- Конструктор - спеціальний метод для створення об'єктів
- Має таку ж назву, як і клас
- Викликається автоматично при створенні об'єкта (`new Student(...)`)
- `this.` - вказує на поля поточного об'єкта
- Використовується `this`, бо параметри мають такі ж назви, як і поля

**Приклад створення об'єкта:**
```java
Student student1 = new Student("Олександр", "Коваленко", 2, 85.5, "Комп'ютерні науки");
```

---

### 4. Геттери (Getters)

```java
public String getName() {
    return name;
}

public int getCourse() {
    return course;
}
```

**Пояснення:**
- Геттери - методи для отримання значень приватних полів
- Починаються з `get` + назва поля з великої літери
- Повертають значення поля (`return`)

**Приклад використання:**
```java
String studentName = student1.getName();
int currentCourse = student1.getCourse();
```

---

### 5. Сеттери (Setters)

```java
public void setCourse(int course) {
    if (course >= 1 && course <= 6) {
        this.course = course;
    } else {
        System.out.println("Помилка: курс має бути від 1 до 6");
    }
}
```

**Пояснення:**
- Сеттери - методи для зміни значень приватних полів
- Починаються з `set` + назва поля з великої літери
- Можуть містити перевірку коректності даних (валідацію)
- `void` - метод нічого не повертає

**Перевага сеттерів:**
- Можна перевірити правильність даних перед зміною
- Наприклад, курс може бути тільки від 1 до 6

**Приклад використання:**
```java
student1.setCourse(3);  // Коректно
student1.setCourse(10); // Виведе помилку
```

---

### 6. Додаткові методи

#### 6.1. Метод displayInfo()

```java
public void displayInfo() {
    System.out.println("\n=== Інформація про студента ===");
    System.out.println("ПІБ: " + surname + " " + name);
    System.out.println("Факультет: " + faculty);
    System.out.println("Курс: " + course);
    System.out.println("Середній бал: " + averageGrade);
}
```

**Пояснення:**
- Виводить всю інформацію про студента в зручному форматі
- Використовує значення полів об'єкта

---

#### 6.2. Метод getScholarshipStatus()

```java
public String getScholarshipStatus() {
    if (averageGrade >= 90) {
        return "Підвищена стипендія";
    } else if (averageGrade >= 75) {
        return "Звичайна стипендія";
    } else if (averageGrade >= 60) {
        return "Без стипендії";
    } else {
        return "Академічна заборгованість";
    }
}
```

**Пояснення:**
- Визначає статус стипендії на основі середнього балу
- Повертає рядок (`String`) з результатом
- Використовує каскадні умови `if-else if-else`

---

#### 6.3. Метод promoteToNextCourse()

```java
public void promoteToNextCourse() {
    if (course < 6) {
        course++;
        System.out.println(name + " переведено на " + course + " курс");
    } else {
        System.out.println(name + " вже на останньому курсі");
    }
}
```

**Пояснення:**
- Переводить студента на наступний курс
- `course++` - інкремент (збільшення на 1)
- Перевіряє, чи студент не на останньому курсі

---

#### 6.4. Метод hasBetterGradeThan()

```java
public boolean hasBetterGradeThan(Student other) {
    return this.averageGrade > other.averageGrade;
}
```

**Пояснення:**
- Порівнює середній бал поточного студента з іншим студентом
- Приймає параметр типу `Student` (інший об'єкт)
- Повертає `boolean` (true/false)
- `this.averageGrade` - бал поточного студента
- `other.averageGrade` - бал студента, з яким порівнюємо

**Приклад використання:**
```java
if (student1.hasBetterGradeThan(student2)) {
    System.out.println("Студент 1 має кращий бал");
}
```

## Головний клас Main

### 1. Створення об'єктів

```java
Student student1 = new Student("Олександр", "Коваленко", 2, 85.5, "Комп'ютерні науки");
Student student2 = new Student("Марія", "Шевченко", 3, 92.0, "Інформаційні технології");
```

**Пояснення:**
- `new Student(...)` - виклик конструктора
- Створюється новий об'єкт типу `Student`
- Кожен об'єкт має свої власні значення полів
- `student1` та `student2` - посилання на різні об'єкти

---

### 2. Робота з об'єктами

```java
student1.displayInfo();                              // Виклик методу
String status = student1.getScholarshipStatus();     // Отримання результату
student1.setAverageGrade(88.0);                      // Зміна значення
```

**Пояснення:**
- Доступ до методів об'єкта через крапку (`.`)
- Кожен об'єкт може викликати методи свого класу

---

### 3. Інтерактивне створення студента

```java
System.out.print("Введіть ім'я студента: ");
String name = scanner.nextLine();
// ... інший ввід
Student customStudent = new Student(name, surname, course, grade, faculty);
```

**Пояснення:**
- Користувач вводить дані з клавіатури
- На основі введених даних створюється новий об'єкт
- Демонструє гнучкість класів

## Ключові концепції ООП (продемонстровані в програмі)

### 1. Клас та об'єкт
- **Клас** - шаблон, опис
- **Об'єкт** - конкретний екземпляр класу
- Аналогія: Креслення будинку (клас) → Реальний будинок (об'єкт)

### 2. Інкапсуляція
- Приховування даних (`private`)
- Доступ через методи (геттери/сеттери)
- Захист даних від некоректних змін

### 3. Конструктор
- Ініціалізація об'єкта
- Початкові значення полів
- Викликається при створенні об'єкта

### 4. Методи
- Поведінка об'єкта
- Дії, які може виконувати об'єкт
- Робота з даними об'єкта

### 5. this
- Посилання на поточний об'єкт
- Використовується для розрізнення полів і параметрів

## Переваги ООП (на прикладі програми)

1. **Організація коду:**
   - Логіка студента зібрана в одному класі
   - Код легко читати та підтримувати

2. **Повторне використання:**
   - Можна створювати багато студентів з одного класу
   - Не треба дублювати код

3. **Захист даних:**
   - Поля приватні
   - Зміна тільки через сеттери з перевіркою

4. **Розширюваність:**
   - Легко додати нові методи (наприклад, `isHonorStudent()`)
   - Легко додати нові поля (наприклад, `email`)

## Типові помилки

1. **Забути this в конструкторі:**
   ```java
   // Неправильно:
   name = name;  // Присвоює параметр самому собі
   
   // Правильно:
   this.name = name;  // Присвоює значення полю класу
   ```

2. **Не перевіряти дані в сеттерах:**
   ```java
   // Погано:
   public void setCourse(int course) {
       this.course = course;  // Може бути будь-яке значення
   }
   
   // Добре:
   public void setCourse(int course) {
       if (course >= 1 && course <= 6) {
           this.course = course;
       }
   }
   ```

3. **Робити поля публічними:**
   ```java
   // Неправильно:
   public String name;  // Прямий доступ ззовні
   
   // Правильно:
   private String name;  // Доступ через геттери/сеттери
   ```

## Висновок

Ця лабораторна робота демонструє:
- Створення власних класів
- Інкапсуляцію даних
- Роботу з конструкторами
- Використання геттерів та сеттерів
- Створення та використання методів класу
- Роботу з об'єктами

**Основна ідея ООП:** Програма складається з об'єктів, які взаємодіють один з одним, а не з послідовності інструкцій.
