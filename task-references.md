# VS Code Task References

This document lists all tasks defined in your `.vscode/tasks.json` file, explains what each task does, and provides usage examples.

---

## 1. Compile and Run any .cpp file

**Label:** `Compile and Run any .cpp file`

**Type:** `shell`

**What it does:**
- Compiles the currently open `.cpp` file using `g++`.
- Runs the resulting executable in the same directory.

**How it works:**
- Changes to the file's directory.
- Sets codepage to UTF-8 for proper output.
- Compiles with `g++` and runs the `.exe`.

**Task Definition:**
```json
{
    "label": "Compile and Run any .cpp file",
    "type": "shell",
    "command": "cd \"${fileDirname}\"; chcp 65001 >$null; g++ ${fileBasename} -o ${fileBasenameNoExtension}.exe; .\\${fileBasenameNoExtension}.exe",
    "problemMatcher": [],
    "group": "build",
    "presentation": {
        "echo": true,
        "reveal": "always",
        "focus": false,
        "panel": "shared"
    }
}
```

**How to use:**
1. Open a `.cpp` file in VS Code.
2. Press `Ctrl+Shift+B` or run from Command Palette.
3. Select this task to compile and run.

## 2. Compile and Run any .cpp file in Command panel

**Label:** `Compile and Run any .cpp file in Command panel`

**Type:** `process`

**What it does:**
- Opens a new Command Prompt window.
- Compiles and runs the `.cpp` file, then pauses.

**How it works:**
- Uses `cmd.exe` to start a new window.
- Compiles in the file's directory and runs with pause.

**Task Definition:**
```json
{
    "label": "Compile and Run any .cpp file in Command panel",
    "type": "process",
    "command": "cmd.exe",
    "args": [
        "/c",
        "start cmd.exe /k chcp 65001 ^& cd /d ${fileDirname} ^&^& g++ ${fileBasename} -o ${fileBasenameNoExtension}.exe ^&^& ${fileBasenameNoExtension}.exe ^& pause"
    ],
    "problemMatcher": [],
    "group": "build",
    "presentation": {
        "echo": true,
        "reveal": "always",
        "focus": false,
        "panel": "shared"
    }
}
```

**How to use:**
- Same as above, but runs in a separate Command Prompt window.

## 3. C/C++: g++.exe build active file

**Label:** `C/C++: g++.exe build active file`

**Type:** `shell`

**What it does:**
- Builds the active `.cpp` file using MSYS2's g++ with debug info.

**How it works:**
- Uses `C:\msys64\ucrt64\bin\g++.exe` with diagnostics and debug flags.
- Default build task.

**Task Definition:**
```json
{
    "type": "shell",
    "label": "C/C++: g++.exe build active file",
    "command": "chcp 65001 >$null; C:\\msys64\\ucrt64\\bin\\g++.exe -fdiagnostics-color=always -g ${file} -o ${fileDirname}\\${fileBasenameNoExtension}.exe",
    "options": {
        "cwd": "${fileDirname}"
    },
    "problemMatcher": [
        "$gcc"
    ],
    "group": {
        "kind": "build",
        "isDefault": true
    },
    "detail": "Task generated by Debugger."
}
```

**How to use:**
- Press `Ctrl+Shift+B` to build the active file.

## 4. Analyze and Explain Code (Shell)

**Label:** `Analyze and Explain Code (Shell)`

**Type:** `shell`

**What it does:**
- Runs a PowerShell script to analyze the code and append explanations.

**How it works:**
- Executes `scripts/analyze_and_explain.ps1` with the file as argument.

**Task Definition:**
```json
{
    "label": "Analyze and Explain Code (Shell)",
    "type": "shell",
    "command": "powershell",
    "args": [
        "-NoProfile",
        "-ExecutionPolicy",
        "Bypass",
        "-File",
        "${workspaceFolder}/scripts/analyze_and_explain.ps1",
        "${file}"
    ],
    "options": {
        "cwd": "${workspaceFolder}"
    },
    "presentation": {
        "reveal": "always"
    }
}
```

**How to use:**
- Open a code file and run this task to get analysis appended.

## 5. Start All MCP Servers

**Label:** `Start All MCP Servers`

**Type:** `shell`

**What it does:**
- Starts all MCP (Model Context Protocol) servers in the background.

**How it works:**
- Runs multiple `npx` commands to start various MCP servers.

**Task Definition:**

```json
    "problemMatcher": [],
    "isBackground": true,
    "presentation": {
        "reveal": "always",
        "panel": "shared"
    }
}
```

**How to use:**
- Run this task to start all MCP servers for AI assistance.

## 7. Start All MCP Servers (parallel)

**Label:** `Start All MCP Servers (parallel)`

**Type:** (depends on others)

**What it does:**
- Starts all MCP servers in parallel using dependencies.

**How it works:**
- Depends on individual MCP tasks and runs them in parallel.

**Task Definition:**
```json
{
    "label": "Start All MCP Servers (parallel)",
    "dependsOn": [
        "Start MCP: context7",
        "Start MCP: sequentialthinking",
        "Start MCP: memory",
        "Start MCP: filesystem",
        "Start MCP: qodo"
    ],
    "dependsOrder": "parallel"
}
```

**How to use:**
- Run to start all MCP servers simultaneously.
